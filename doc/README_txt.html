Installing and Running the Application

In order to run the game you have to position yourself inside the directory of the project and then run the file “ecatepec.rb” with the following command:

ruby -I . -w "ecatepec.rb"

Afterwards, point your web brower at the following URL: +localhost:4567/+

Patterns Used

The patterns that were used within our project are:

  • State Pattern: We used this pattern in order to separate the functionality into states and substates that could handle the instances of the game better. The states are instances of Room and are handled in the clases.rb.

  • Model View Controller: The application follows the classical web implementation of the MVC architectural pattern. The models (.rb files) and views (.erb files) are stored in the corresponding models and views directory. The controller is contained in ecatepec.rb file.

  • Strategy Factory: Each of our states behaves differently so we send a Proc with all the code that it must implement. This is handled in the gameFactory class that is contained inside the clases.rb.
